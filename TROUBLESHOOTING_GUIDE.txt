# GitHub Pages 部署问题排查指南

## 404 错误常见原因及解决方案

### 1. GitHub Pages 未启用
**问题**：仓库设置中未启用 GitHub Pages
**解决方案**：
- 访问 https://github.com/LIAMLDH/LIAMLDH.github.io
- 点击 Settings 标签
- 向下滚动到 Pages 部分
- 确保 Source 设置为 "Deploy from a branch"
- 选择 "gh-pages" 分支和 "/ (root)" 文件夹
- 点击 Save

### 2. GitHub Actions 构建未完成
**问题**：构建过程仍在进行中
**解决方案**：
- 访问仓库的 Actions 标签页
- 检查是否有正在运行或失败的工作流
- 如果构建失败，查看日志找出错误原因
- 等待构建完成后再访问网站

### 3. 构建配置问题
**问题**：构建配置不正确
**检查点**：
- 确认 vite.config.js 中的 base 设置为 "/"
- 确认 .github/workflows/deploy.yml 存在且配置正确

### 4. 缺少 gh-pages 分支
**问题**：GitHub Actions 未创建 gh-pages 分支
**解决方案**：
- 检查仓库是否包含 gh-pages 分支
- 如果没有，可能是 GitHub Actions 构建失败
- 检查 Actions 日志查找问题

### 5. 仓库命名错误
**问题**：仓库名称不正确
**解决方案**：
- 确保仓库名称为 LIAMLDH.github.io（其中 LIAMLDH 是您的 GitHub 用户名）
- 仓库名称必须与 GitHub Pages 地址完全匹配

## 验证步骤

### 第一步：检查仓库设置
1. 访问 https://github.com/LIAMLDH/LIAMLDH.github.io
2. 点击 Settings > Pages
3. 确认 Source 设置为 "Deploy from a branch"
4. 确认分支设置为 "gh-pages"

### 第二步：检查构建状态
1. 点击仓库的 Actions 标签
2. 查找最近的 "Deploy to GitHub Pages" 工作流
3. 确认其状态为 "Success"（绿色对勾）

### 第三步：检查分支
1. 点击仓库的 Code 标签
2. 检查分支下拉菜单中是否有 "gh-pages" 分支
3. 确认该分支存在

### 第四步：等待生效
1. GitHub Pages 首次部署可能需要几分钟时间
2. 刷新页面后等待 5-10 分钟再尝试访问

## 强制刷新浏览器缓存
有时浏览器会缓存旧页面，尝试以下操作：
- 按 Ctrl+F5 强制刷新页面
- 或使用 Ctrl+Shift+Delete 清除浏览器缓存
- 或在隐私/无痕模式下访问网站

## 检查构建输出
如果以上步骤都无法解决问题：
1. 访问仓库的 gh-pages 分支
2. 检查是否存在 dist 目录中的构建文件
3. 确认 index.html 文件存在于根目录

## 联系支持
如果问题持续存在，请检查：
- GitHub Status 页面 (https://www.githubstatus.com/) 确认服务正常
- GitHub 社区论坛寻求帮助
