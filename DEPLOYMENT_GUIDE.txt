# 学生选课管理系统部署指南

## 重要提示

GitHub Pages 只能托管静态文件，无法直接运行 Spring Boot 后端服务。本项目包含前后端分离架构，需要分别部署。

## 部署选项

### 选项一：仅前端部署到 GitHub Pages（演示目的）

1. 前端 Vue 应用已配置自动部署到 GitHub Pages
2. 需要单独的后端 API 服务（见下方后端部署说明）

### 选项二：完整部署（推荐）

- 前端：GitHub Pages
- 后端：Heroku、Railway 或其他 Java 支持的云平台

## 前端部署到 GitHub Pages

### 自动部署（已配置）

1. 确保你的 GitHub 仓库名为 `LIAMLDH.github.io`
2. 推送代码到 main 分支：
   ```
   git add .
   git commit -m "Update configuration for GitHub Pages deployment"
   git push origin main
   ```

3. GitHub Actions 将自动构建并部署到 GitHub Pages
4. 访问 `https://LIAMLDH.github.io/`

### 手动部署

1. 构建项目：
   ```
   cd student-vue
   npm install
   npm run build
   ```

2. 将 `dist` 目录中的内容上传到 GitHub Pages

## 后端部署

由于 GitHub Pages 无法运行 Spring Boot 应用，你需要将后端部署到其他平台：

### 部署到 Heroku

1. 创建 `system.properties` 文件指定 Java 版本：
   ```
   java.runtime.version=17
   ```

2. 修改 `application.properties` 中的数据库连接信息为 Heroku 提供的数据库

3. 使用 Heroku CLI 部署：
   ```
   heroku create
   git push heroku main
   ```

### 部署到 Railway

1. 在 Railway 上创建新项目
2. 连接你的 GitHub 仓库
3. 配置构建和启动命令

## API 代理配置

在开发环境中，Vite 配置了代理：
- 前端请求 `/api/*` 将被代理到 `http://localhost:8080`
- 生产环境中，需要将 API 请求指向实际的后端服务地址

## 生产环境 API 地址配置

在生产环境中，您需要更新前端 API 请求的基础 URL：

修改 `student-vue/src/api/request.js` 中的 baseURL 为你的后端服务地址

## 注意事项

1. 数据库需要单独配置和部署
2. 生产环境应使用安全的密码和密钥
3. 考虑使用环境变量存储敏感信息
4. 确保 CORS 配置允许来自 GitHub Pages 的请求

## 故障排除

如果前端无法连接后端 API：
1. 检查后端服务是否正常运行
2. 验证 API 地址是否正确
3. 确认 CORS 设置
4. 查看浏览器开发者工具中的网络错误

## 部署脚本

项目中包含了两个部署脚本：
- `deploy.bat` - Windows 批处理脚本
- `deploy.sh` - Linux/Mac Shell 脚本

运行相应的脚本可以自动执行部署过程。
